# Requires Python >=3.11. Vercel's default Python runtime is 3.12, so this
# is fine there. NOT installable in the local build sandbox (Python 3.10) -
# could not be pip-installed or exercised locally; verify after deploying.
#
# ROOT CAUSE OF THE EARLIER "ImportError: cannot import name 'OKXAuthConfig'
# from 'x402.http'" FAILURE (found by re-reading OKX's actual Python SDK
# docs at web3.okx.com/onchainos/dev-docs/payments/sdk-python, which
# documents OKXAuthConfig/OKXFacilitatorClient/OKXFacilitatorConfig as real,
# current, first-class exports of okxweb3-app-x402 - our original code in
# app/x402.py was correct all along):
#
# okxweb3-app-x402 VENDORS its own copy of the x402 protocol library at
# `_vendor/x402/...` (visible in the original error's traceback path:
# ".../_vendor/x402/http/__init__.py"). We separately added a top-level
# `x402[evm]` dependency to fix an unrelated "EVM mechanism requires
# ethereum packages" error - but that pulled in Coinbase's own generic,
# separately-published `x402` PyPI package, which installs its own
# `x402.http` module WITHOUT the OKX-specific classes. Depending on import
# resolution order, that shadowed okxweb3-app-x402's vendored copy, which
# is exactly why OKXAuthConfig went missing.
#
# Fix: request the `evm` extra FROM okxweb3-app-x402 itself (it ships its
# own `evm`/`fastapi` extras - see PyPI "Provides-Extra") instead of
# installing a second, conflicting top-level x402 package.
okxweb3-app-x402[evm,fastapi]==0.1.1
